üü¢ What is NodeJS?

NodeJS Definition

NodeJS is a JavaScript runtime built on Chrome's V8 JavaScript engine that allows you to run JavaScript code outside of a web browser.

Why Use NodeJS?

  • Server-side JavaScript: Use the same language for frontend and backend
  • Fast and Scalable: Non-blocking I/O and event-driven architecture
  • Large Ecosystem: Massive npm package repository
  • Cross-platform: Runs on Windows, macOS, and Linux
  • Real-time Applications: Perfect for chat apps, APIs, and microservices

‚úÖ Great for:

  • REST APIs
  • Real-time applications
  • Microservices
  • Command-line tools
  • Build tools

‚ùå Not ideal for:

  • CPU-intensive tasks
  • Complex mathematical operations
  • Heavy computational work

Zero-Dependencies NodeJS Program

What does "Zero Dependencies" mean?

A program that only uses NodeJS built-in modules without requiring external npm packages.

File System Example

// file-reader.js
const fs = require('fs');
const path = require('path');

// Read current directory
const currentDir = process.cwd();
console.log(`Current directory: ${currentDir}`);

// List files in directory
fs.readdir(currentDir, (err, files) => {
    if (err) {
        console.error('Error reading directory:', err);
        return;
    }
    
    console.log('\nFiles in current directory:');
    files.forEach(file => {
        const filePath = path.join(currentDir, file);
        const stats = fs.statSync(filePath);
        const type = stats.isDirectory() ? 'DIR' : 'FILE';
        console.log(`${type}: ${file}`);
    });
});

  • os - Operating system
  • crypto - Cryptography
  • url - URL parsing
  • util - Utilities
  • events - Event emitter

Creating a Simple Web Server

// server.js
const express = require('express');
const axios = require('axios');

const app = express();
const PORT = 3000;

// Middleware to parse JSON
app.use(express.json());

// Route to get joke from external API
app.get('/joke', async (req, res) => {
    try {
        const response = await axios.get('https://official-joke-api.appspot.com/random_joke');
        const joke = response.data;
        
        res.json({
            setup: joke.setup,
            punchline: joke.punchline,
            timestamp: new Date().toISOString()
        });
    } catch (error) {
        res.status(500).json({ error: 'Failed to fetch joke' });
    }
});

// Start server
app.listen(PORT, () => {
    console.log(`Server running on http://localhost:${PORT}`);
    console.log(`Try visiting: http://localhost:${PORT}/joke`);
});
node server.js # Server running on http://localhost:3000 # Try visiting: http://localhost:3000/joke

üèÉ‚Äç‚ôÇÔ∏è What is a Process?

Process Definition

A process is a running instance of a program. When you execute a program, the operating system creates a process to run that program.

Program vs Process

üìÑ Program

  • Code stored on disk
  • Static file
  • Instructions waiting to be executed
  • Example: firefox binary

‚ö° Process

  • Program loaded into memory
  • Dynamic, active instance
  • Has Process ID (PID)
  • Example: Firefox running with PID 1234

Process Components

  • Process ID (PID): Unique identifier assigned by the OS
  • Memory Space: Code, data, stack, heap areas
  • File Descriptors: Open files and network connections
  • Environment: Variables and working directory
  • State: Running, waiting, stopped, etc.
ps aux | head -5 # USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND # root 1 0.0 0.1 167648 11504 ? Ss 08:00 0:01 /sbin/init # root 2 0.0 0.0 0 0 ? S 08:00 0:00 [kthreadd] # john 1234 2.1 3.4 342432 67584 ? Sl 09:15 0:45 firefox

Process Creation: ls | grep '[A-Z]'

What processes are created?

When you run ls | grep '[A-Z]', the shell creates TWO separate processes connected by a pipe.

Shell Process
creates
ls Process
|
grep Process

Step-by-Step Breakdown

  1. Shell parses command: Recognizes pipe operator |
  2. Creates ls process: Runs /bin/ls program
  3. Creates grep process: Runs /bin/grep program
  4. Sets up pipe: Connects ls stdout to grep stdin
  5. Processes run: ls lists files, grep filters them
  6. Cleanup: Both processes terminate, shell continues

üìã ls Process

  • Program: /bin/ls
  • Function: List directory contents
  • Output: Goes to pipe (not terminal)

üîç grep Process

  • Program: /bin/grep
  • Function: Filter lines matching pattern
  • Input: Comes from pipe
  • Output: Goes to terminal

üîß System Calls (syscalls)

System Call Definition

A system call (syscall) is a request made by a program to the operating system kernel to perform a privileged operation on behalf of the program.

Why Do We Need System Calls?

  • Hardware Protection: User programs can't directly access hardware
  • Resource Management: Kernel controls access to files, network, etc.
  • Security: Prevents programs from interfering with each other
  • Abstraction: Provides consistent interface across different hardware

üîç Finding Process IDs with ps

What is a PID?

A Process ID (PID) is a unique number assigned by the operating system to identify each running process.

Essential ps Commands

# Show all processes with full details
ps aux

# Show your processes only
ps -u $USER

# Show process tree
ps -ef --forest

# Find specific process
ps aux | grep firefox

# Show processes by name

# Show detailed info for specific PID
ps -p 1234 -o pid,ppid,cmd,%cpu,%mem

üìÅ File Descriptors Explained

File Descriptor Definition

A file descriptor (FD) is a number that the operating system assigns to each open file, socket, or pipe in a process. It's like a "handle" to access the resource.

Standard File Descriptors

üî¢ The Big Three

  • 0 (stdin): Standard input
  • 1 (stdout): Standard output
  • 2 (stderr): Standard error

üìà Custom File Descriptors

  • 3, 4, 5...: Files opened by program
  • Each open() call gets next available number
  • Released when file is closed

File Descriptor Example

# fd_example.py
import os

print(f"PID: {os.getpid()}")
print("Opening files...")

# These will get file descriptors 3, 4, 5
file1 = open("test1.txt", "w")
file2 = open("test2.txt", "r") 
file3 = open("test3.txt", "a")

print(f"File 1 FD: {file1.fileno()}")  # Usually 3
print(f"File 2 FD: {file2.fileno()}")  # Usually 4
print(f"File 3 FD: {file3.fileno()}")  # Usually 5

input("Press enter to close files...")
file1.close()
file2.close() 
file3.close()

üí° Fun Facts

  • File descriptors are per-process (each process has its own set)
  • They're reused when files are closed
  • Network sockets also use file descriptors!

üî¨ System Call Tracing with strace

What does strace do?

strace intercepts and displays system calls made by a process, showing you exactly how a program interacts with the kernel.

Basic strace Commands

# Trace all system calls of a command
strace ls

# Trace specific system calls only
strace -e trace=open,read,write ls

# Trace a running process by PID
strace -p 1234

# Save output to file
strace -o trace.log ls

# Show timestamps
strace -t ls

# Follow child processes
strace -f ./parent_process

Sample strace Output

$ strace -e trace=openat,read,write echo "hello"

openat(AT_FDCWD, "/lib/x86_64-linux-gnu/libc.so.6", O_RDONLY|O_CLOEXEC) = 3
read(3, "\177ELF\2\1\1\3\0\0\0\0\0\0\0\0...", 832) = 832
write(1, "hello\n", 6) = 6

üìñ Reading strace Output

  • openat(...): System call name and arguments
  • = 3: Return value (file descriptor)
  • read(3, ...): Read from FD 3
  • write(1, ...): Write to FD 1 (stdout)

üéØ Common System Calls

  • openat/open: Open files
  • read/write: File I/O
  • close: Close files
  • mmap: Memory mapping
  • clone/fork: Create processes

üßê Interpreting strace Output

Understanding Return Values

‚úÖ Success Cases

  • Positive numbers: Success (often file descriptors or bytes read/written)
  • Zero: Success (often end-of-file for read())

‚ùå Error Cases

  • -1: Error occurred
  • ENOENT: File not found
  • EACCES: Permission denied
  • EBADF: Bad file descriptor

Real Example: File Operations

# Python script doing file operations
openat(AT_FDCWD, "data.txt", O_RDONLY) = 3        # Success: FD 3
read(3, "Hello World\n", 4096) = 12              # Read 12 bytes
read(3, "", 4096) = 0                           # End of file
close(3) = 0                                    # Success

openat(AT_FDCWD, "missing.txt", O_RDONLY) = -1 ENOENT (No such file or directory)

Using man 2 for Documentation

man 2 open # Manual for open() system call man 2 read # Manual for read() system call man 2 write # Manual for write() system call

üí° Pro Tips

  • man 2: System call documentation
  • man 3: Library function documentation
  • Filter noise: Use -e trace=file to see only file operations
  • Timestamps: Use -t to see when calls happen
